## 一、系统架构

```
run_backtest.py（回测入口）
        ↓
VolumeStrategyBacktest（事件驱动回测引擎）
        ↓
┌──────────────────────────────────────┐
│   信号层                              │
│   VolumeQuantizer  → 量能指标计算     │

└──────────────────────────────────────┘
        ↓

        ↓
┌──────────────────────────────────────┐
│   分析层                              │
│   BacktestAnalyzer → 结果分析导出    │
└──────────────────────────────────────┘
```

## 二、回测引擎核心设计

### 关键原则（防止回测偷看未来）

| 原则 | 实现方式 |
|------|----------|
| T日收盘生成信号 | 信号基于T日收盘数据计算 |
| T+1日开盘执行 | 买入在T+1日开盘价执行 |
| 滑点模拟 | 买入+0.05%，卖出-0.05% |
| 真实交易成本 | 佣金0.03%双向+印花税0.1%卖出 |

### 交易成本配置

```python
COMMISSION_RATE = 0.0003   # 佣金：双向 0.03%
STAMP_TAX_RATE  = 0.001    # 印花税：0.1%，仅卖出收取
SLIPPAGE_RATE   = 0.0005   # 滑点：0.05%
```

### 仓位管理

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `max_positions` | 5 | 最大同时持仓数 |
| `max_single_position_pct` | 20% | 单只最大仓位比例 |
| `min_signal_strength` | 50 | 最小买入信号强度 |
| `max_risk_score` | 50 | 最大允许风险评分 |

## 三、第一阶段改进：ATR动态止损

### 改进背景

原始模型使用固定百分比止损（-3%），存在以下问题：
- 高波动股票：止损太紧，被噪音止损
- 低波动股票：止损太宽，亏损过大
- 盈亏比0.93（平均亏损 > 平均盈利）

### ATR动态止损原理

ATR（Average True Range，平均真实波幅）根据股票实际波动性动态计算止损：

```
True Range = max(
    high - low,              # 当日振幅
    abs(high - close_prev),  # 跳空高开
    abs(low - close_prev)    # 跳空低开
)
ATR = SMA(True Range, 14天)

止损价 = 买入价 - 1.5 × ATR_multiplier × ATR
目标价 = 买入价 + 3.0 × ATR
```

### 追踪止损（Trailing Stop）

```
追踪止损价 = 持仓以来最高价 - 1.5 × ATR
（但不低于初始止损价）
```

当股价上涨时，止损价自动上移，锁定利润。

### 止损策略参数

| 参数 | 默认值 | 含义 |
|------|--------|------|
| `use_atr_stop_loss` | True | 启用ATR动态止损 |
| `atr_period` | 14 | ATR计算周期 |
| `atr_multiplier` | 2.0 | ATR倍数 |
| `use_trailing_stop` | True | 启用追踪止损 |
| `time_stop_days` | 10 | 时间止损天数（原5天→改10天）|

## 五、运行方式

### 基本回测

```bash
# 回测指定时间段（默认股票池46只）
python run_backtest.py --start 2021-01-01 --end 2025-12-31

# 全量主板前100只
python run_backtest.py --start 2024-01-01 --end 2025-12-31 --full-market --top-n 100

# 指定股票池
python run_backtest.py --start 2024-01-01 --stocks 600519,000001,600036
```

### ATR止损参数调整

```bash
# 启用ATR止损（默认启用）
python run_backtest.py --start 2024-01-01 --use-atr-stop

# 调整ATR参数（更紧凑的止损）
python run_backtest.py --start 2024-01-01 --atr-multiplier 1.5

# 禁用ATR止损，使用原始固定止损
python run_backtest.py --start 2024-01-01 --no-atr-stop

# 禁用追踪止损
python run_backtest.py --start 2024-01-01 --no-trailing-stop
```

### 完整参数列表

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--start` | 必填 | 回测开始日期 |
| `--end` | 今天 | 回测结束日期 |
| `--capital` | 1,000,000 | 初始资金 |
| `--strategy` | balanced | 策略（aggressive/balanced/conservative）|
| `--max-positions` | 5 | 最大持仓数 |
| `--max-pos-pct` | 0.20 | 单股最大仓位比例 |
| `--min-signal` | 50 | 最小信号强度 |
| `--max-risk` | 50 | 最大风险评分 |
| `--time-stop` | 10 | 时间止损天数 |
| `--use-atr-stop` | True | 启用ATR动态止损 |
| `--atr-period` | 14 | ATR计算周期 |
| `--atr-multiplier` | 2.0 | ATR倍数 |
| `--use-trailing-stop` | True | 启用追踪止损 |
| `--full-market` | False | 全量主板模式 |
| `--top-n` | 100 | 全量模式取前N只 |

## 六、数据缓存系统

### 本地缓存（优先使用）

```
tools/output/data_cache/{code}_local.parquet
```

由 `tools/build_stock_cache.py` 生成，包含完整历史数据。
使用前需更新缓存到最新日期：

```bash
python tools/build_stock_cache.py
```

### 临时缓存

```
data_cache/{code}_{start}_{end}.parquet
```

按日期范围缓存，无需手动管理。
